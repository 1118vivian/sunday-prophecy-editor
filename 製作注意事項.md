# 製作注意事項

## 版面與輸出

- 預覽、PDF、Word 應盡量使用同一份 A4 固定版面，避免三種輸出長得不一樣。
- PDF 不要直接截取螢幕上的預覽區，因為電腦或手機視窗寬度不同會造成比例跑掉。
- PDF 目前使用固定 A4 隱藏版面輸出，尺寸是 `794 x 1123`，背景統一白色。
- Word 手機版會重新排版一般表格，可能造成表格被擠到左邊；因此目前 Word 採固定 A4 圖片式版面，手機顯示比較穩定。
- Word 檔內容若要修改，應回到網頁欄位修改後重新產生，不建議在 Word 裡直接編輯。

## 手機與電腦差異

- `分享 Word` 只適合手機使用，因為手機可叫出系統分享面板並選 LINE。
- 電腦版不顯示 `分享 Word` 按鈕，保留 `產生 DOCX` 和 `產生 PDF`。
- 瀏覽器不一定支援直接分享檔案；若不支援，程式會下載 Word，讓使用者再到 LINE 選檔案傳送。

## 檔案產生套件

- `docx.umd.js` 用來產生 Word。
- `html2canvas.min.js` 用來把固定 A4 版面轉成圖片。
- `jspdf.umd.min.js` 用來產生單頁 PDF。
- 這些套件放在 `vendor/`，避免依賴外部 CDN，沒有網路時也比較穩。

## 快取與版本號

- 修改 `app.js` 後，要更新 `index.html` 裡的 `app.js?v=...`。
- 修改 `styles.css` 後，要更新 `index.html` 裡的 `styles.css?v=...`。
- GitHub Pages 和手機瀏覽器可能會快取舊檔，版本號可以強制使用新版。

## GitHub Pages

- 網站網址是 `https://1118vivian.github.io/sunday-prophecy-editor/`。
- 推上 `main` 後，GitHub Actions 會自動部署 Pages。
- 推送後通常要等 1 到 2 分鐘才會更新線上版。

## 測試重點

- 產生 DOCX 後，要用手機打開檢查 A4 版面是否完整。
- 產生 PDF 後，要確認只有 1 頁，且比例自然、背景一致。
- 修改按鈕或手機版功能時，要同時檢查手機寬度下是否換行正常。
- 修改欄位文字或提示文字時，要確認預覽與檔名沒有受影響。

## Git 記錄

- 之後所有專案的 commit 訊息都使用中文簡易說明。
- 例：`調整PDF版面`、`新增Word分享功能`、`修正手機顯示`。
